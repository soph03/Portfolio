Overview

Introduction

EA is one of the largest gaming companies in the world, including games such as Apex Legends, The Sims 4, and It Takes Two. During my 4 month internship, I worked on the EA App, a PC platform for players to download and manage their games, as well as connect with other players across platforms.

Project

As a UX Design Intern, I discovered problems, created new game states and simplified the game tile system. Throughout the process, I worked with my Manager Dave, gaining experience presenting, building rationale, and practicing product thinking.

Problem

1. Tiles are being presented at the same level of importance.

2. The price component of a discounted game tile is visually clustered.

2. Add-on tiles are represented inconsistently throughout the application.

Context

EA Application

The EA App is a windows application for players to download, update and manage their games. It provides the ability to connect with friends and integrate with external applications such as discord.

Throughout the application, game tiles can be found displaying key information such as the image of the game, price, and membership type.

Design Process

Audit/User research

Pt 1. Problem Identification

To familiarize myself with game tiles and application, I conducted an audit to identify the different types of tiles in the app and read through previous User Research reports.

I then sorted the tiles into groups(user journey, location, tile size, and archetype motivations to identify potential inconsistencies and gaps.

Pt 2. Problem prioritization

Along with my manager, the problems we identified were sorted and organized from highest impact to least effort to determine which problems to focus on.

Top 3 Focus Area

1. How might we present different game states?
  • 2. How might we place price and discounts more effectively?
  • 3. How might we re-imagine a consistent Add-on tile design?

    • User Research

    How might we present different game states?

    Competitive Analysis

    With the problem space in mind, I wanted to understand how other competitors presented different states and the copy used, as well as how tiles are differentiated between games and DLC(game add-ons).

    Findings

    1. Some apps have categorized pages: new, top seller, top rated, upcoming, etc
    2. States identified through badges, banners, or labels
    3. DLC identified within the image, title, secondary text, or badges

    Design Space

    How might we present different game states while leveraging and minimizing existing design systems.

    Existing Designs

    Design Components

    A game tile consists of multiple components. In addition to exploring different tile sizes and hues of brand colours, the components I want leverage are:

    1. Secondary badge
    2. Secondary text
    3. Label

    Success Criteria

    1. Create a clear distinction between tiles with states and those without
    2. Create a clear distinction between game tiles and add-ons
    3. Reduce complexity within the game tile system
    4. Leverage existing design components

    Design Considerations

    Copy Ideas

    I looked through the user journey of a player, states used by competitor's and states that are in the process of development to determine a list of possible ideas.

    After discussing with my Manager, the game states we chose to focus on were: Coming soon, Pre-orderable, New, Discount percentage (ex. -30%), and In-collection.

    Information Hierarchy

    Although the position of the badge and the secondary text is fixed, there is some flexibility on where we want to place the tag. Through some exploring, I wanted to have the 'New' label placed in front of the game title because it provides separation between the other labels within the tile, clear hierarchy, and does not affect the title when the text wraps.

    Visual Hierarchy

    Due to sales being the main driver of revenue, the discount state respectively lead in visual hierarchy, followed by the 'New' state, where I learned from user interviews and competitive analysis that new games is a key motivator for players. Lastly, we have the rest of the states incorporating a muted visual.

    Design Revisions

    First Draft

    A few designs were brought to the greater design team and it was agreed that this design could be pursued further because the separation between the badges and labels helped create stronger visual hierarchy.

    However, one issue that we encountered is the visual noise produced by the 4 different labels within the add-on tile.

    How can we minimize the design of the add-on tile?

    First design change

    Remove strike through price

    Second design change

    RemoveAdd-on label and differentiating Add-ons through shape and a shorter tile size.

    Third design change

    Replace pre-orderable with coming soon

    Fourth design change

    Create a system and hierarchy where only one game state appears at a time

    Game States Final Design

    Brief

    Currently, game titles are being presented at the same level of importance/hierarchy. This creates a design opportunity to draw increased attention to specific titles in a discovery setting that occupy a desirable or contextually useful status for the user.

    What is a game state?

    A certain stage within the life cycle of a game product. The game state that we have designed are split into two groups.

    Success criteria

    1. Create a clear distinction between tiles with states and those without
    2. Create a clear distinction between game tiles and add-ons
    3. Reduce complexity within the game tile system
    4. Leverage existing design components

    Final Polish

    Goals of game state:

    To Attract

    Unreleased Game

    Prior to the release of a game, there are two states: Coming soon and Pre-orderable.

    Games that are unreleased are denoted with a 'Coming soon' label and if they are orderable, a price tag is included in addition to the label.

    New Game

    When a game is released, the game tile would include the 'New' label to notify and attract players.

    Existing Game

    On special occasions and holidays, EA offers games at a discounted price. During this time, selected games will have a discounted percentage to reflect the change.

    Goals of game state:

    To Inform

    Purchased Game

    After a game is purchased, an 'In Collection' label is located under the tile to signify that the game has been added to their collection and is ready to be played.

    Design Priority

    Each state takes different priority from each other. When states are in conflicting situations, the 'In Collection' label on the very left takes the highest priority, followed by the discount percentage, 'New', and 'Coming soon' label.

    Tile System Simplifications

    1. Displaying add-ons in consistent form factor and in context
    2. Distinguishing games and add-ons through different tile size
    3. Removing the add-on badge
    4. Removing the original strikethrough price
